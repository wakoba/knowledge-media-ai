# Atlas Video Plan

Date: 2026-07-04

## Title

紙一枚が機械になる：折り紙・切り紙がひらく展開構造とソフトロボット

## Video Concept

A4一枚の紙からはじめて、折り紙・切り紙の幾何学が“機械のふるまい”を生み出すことを直感と物語で伝える。ミウラ折りの一方向展開や切り紙の隠れヒンジ、ソフトロボットや宇宙・医療への広がりを、やさしい実験と設計思考（逆設計）の流れで紹介する。

## Estimated Duration

10 minutes

## Visual Style

白背景の手元実写＋シンプルな線画の図解アニメーション＋最小限のモーショングラフィックス。色覚多様性に配慮した配色（山折り＝実線＋カラー、谷折り＝破線＋別カラー）。質感はマットでクリーン。効果音は紙の擦れ・空気の吐出など静かな実音を中心に。

## Target Viewer

科学や工学に興味のある一般の視聴者・中高生・メーカーやデザイナー・宇宙／医療／ロボット分野の最新動向を知りたい人

## Opening Visual

机上のA4を片手で“ひと引き”。ミウラ折りが一斉に開く様子を真上カメラで捉え、同時に縦方向の矢印が横方向に連動して広がる簡潔なオーバーレイ。タイトルが紙面上にタイポグラフィで現れる。

## Scene Plan

### Scene 1: フック：ひと引きで動く“面全体”

Narration Summary:

A4を端から引くだけで一斉に開く不思議。縦に引っ張ると横にも広がる現象が、実は展開構造ややさしいロボットの原理とつながっていることを提示する。

Visual Description:

トップビューの手元実写。紙面上に縦矢印→横矢印のアニメ、軽いパタパタ効果音。引き終わりでタイトルが現れる。

On-screen Text:

ひと引きで一斉展開・縦に引くと横にも広がる？・紙が“機械”になる

Asset Type:

手元実写＋矢印図解

Editing Notes:

0:00コールドオープン→0:08でタイトル。タイトル後は素早いカットで次の導入へ。

Safety Notes:

紙の端で手を切らないよう注意。作業は安定した平面で行う。

Narration Summary:

折り紙と切り紙の工学を入口に、展開構造からソフトロボット、宇宙・医療の応用、そして逆設計の考え方までを一緒にたどる予告。

Visual Description:

A4・金属箔・薄膜のモーフィング。キーワードを紙面上にタイポアニメで配置。

On-screen Text:

折り紙／切り紙・展開構造・ソフトロボット・宇宙／医療・逆設計

Asset Type:

モーショングラフィックス

Editing Notes:

テンポは軽快に。キーワード出しは1語0.6秒間隔。

### Scene 3: なぜ“紙の幾何学”が重要なの？

Narration Summary:

折りや切りのパターンは、材料を変えずに剛性や伸び・曲がりの方向を決められる。小さく運んで大きく広げる展開構造は宇宙・防災・医療に直結し、紙実験は教育や試作の高速サイクルにも向く。

Visual Description:

折畳み→現場で大きく展開の対比アニメ。同じ材料でパターンだけ変えた試片を並べ、硬さや曲がり方の違いを比較する手元演出。

On-screen Text:

幾何が機能を決める・小さく運び大きく広げる・教育／試作に最適

Asset Type:

図解アニメーション＋比較手元実写

Editing Notes:

左右分割画面で“同素材・異パターン”を同時比較。テロップは最小限。

### Scene 4: ミウラ折り：一方向で全体が動く

Narration Summary:

平行四辺形タイルのミウラ折りは理想的には一つの自由度で連続開閉し、端を引くだけで面全体が動く。設定次第で負のポアソン比が現れ、縦に引くと横にも広がる挙動が得られる。

Visual Description:

ミウラ折りの手元実写デモ。パネル＝剛体・折り目＝ヒンジの骨格アニメを重ね、縦→横の連動を矢印で可視化。

On-screen Text:

ミウラ折り・一自由度（理想条件）・端を引くだけで開閉・負のポアソン比（設定次第）

Asset Type:

手元実写＋骨格CG＋矢印図解

Editing Notes:

開閉は正面→斜め→トップの3アングルで短く。フットノートで“理想条件”を小さく表示。

Safety Notes:

紙の角での擦り傷に注意。無理な力を加えない。

### Scene 5: 切り紙：スリットが生む“隠れヒンジ”

Narration Summary:

微小スリットがヒンジとして働き、硬い薄膜にも大きな伸びや曲率を与える。スリットの向き・密度・並べ方で柔らかくする方向や曲げ位置を設計できる。

Visual Description:

縦スリット格子を引っ張る接写。異なるパターンで曲率が変わる比較。スリット周りの応力回り込みを等高線風で可視化。

On-screen Text:

スリット＝隠れヒンジ・向き／密度で柔らかさ設計・大きな伸びと曲率

Asset Type:

手元実写＋等高線風アニメ

Editing Notes:

マクロ寄りで紙繊維の動きを見せる。比較は三分割で同尺。

Safety Notes:

スリット作成時はカッターやハサミの取扱いに注意。カッターマットを使用し手指を刃先から離す。

### Scene 6: “剛さ”をプログラムする：ヒンジとパネルの分業

Narration Summary:

よく曲がるヒンジ部と形を保つパネル部を分けて設計する。薄くする・柔らかい材料を挿入する・テーパをつけるなどの工夫でコンプライアントヒンジを実現し、繰り返しや誤差に強くする。厚くなるほど曲げにくいため、生きヒンジ設計が鍵。

Visual Description:

ヒンジ部だけ半透明で強調した分解CG。厚み違いによる曲げやすさの簡易ベンディング実験。テーパーヒンジ断面のスケッチ。

On-screen Text:

ヒンジ／パネルの分業・コンプライアントヒンジ・厚みと曲げ剛性

Asset Type:

CG分解図＋手元実験スローモーション

Editing Notes:

CG→実験→断面スケッチの順で理解を積み上げる。数値表示は行わず相対差のみ提示。

### Scene 7: どう動かす？ 空気・熱・素材の“内なる力”

Narration Summary:

自動展開や把持にはアクチュエータが必要。空気圧や真空、熱応答や湿度・pH応答の材料、磁場で動く複合材料など用途に応じて選ぶ。蛇腹やキレスリング型の折りは空気圧でなめらかに動き、壊れやすい対象にやさしい。

Visual Description:

空気圧でふくらむ蛇腹が柔らかく対象を掴むデモ。外部刺激（温度・水滴・磁石）のアイコンと形状変化の対応をパネルで表示。

On-screen Text:

空気圧／真空・熱／湿度／pH・磁場・やさしい把持

Asset Type:

手元実写デモ＋アイコン図解

Editing Notes:

対象物は壊れやすいものを静かに提示。圧力数値は表記せず挙動のみ見せる。

Safety Notes:

空気圧は低圧で安全に。加圧中は顔を近づけない。加熱や化学試薬の実演は行わず、該当箇所は図解のみとする。

### Scene 8: 設計の流れ：コンピュテーショナル折り紙で“逆算”

Narration Summary:

目的（展開後の形・必要剛性・把持力の上限など）を定め、タイル角度やスリット密度などの設計空間を設定。剛体折りの運動学や有限要素で力学をシミュレーションし、最適化で候補を絞る。材料・加工を選び、試作と実験で特性を同定しながら更新する。

Visual Description:

目的→設計空間→シミュレーション→最適化→試作→実験→更新のフローチャート。山折り・谷折りを色分けしたパターンが自動生成されるUI風アニメ。紙プロトタイプと解析結果のオーバーレイ同期表示。

On-screen Text:

逆算フロー：目的→設計空間→シミュレーション→最適化→試作→実験→更新

Asset Type:

フローチャートMG＋UI風アニメ＋実写オーバーレイ

Editing Notes:

各ステップ1.5〜2秒でテンポ良く。専門用語は短い補足をツールチップ風に表示。

### Scene 11: 未来への布石：紙から“賢い表面”へ

Visual Description:

## Subtitle Direction

日本語字幕。1行あたり全角14〜16字、最大2行。専門用語は初出で短い補足を括弧で併記（例：ポアソン比）。強調は色＋太字ではなく白文字＋半透明黒帯で可読性重視。効果音が入る場面は字幕を0.2秒早出し。脚注系の注意（理想条件・模型表現など）は小さめフォントで画面下に数秒表示。

## Narration Direction

落ち着いたトーンで、驚きよりも“仕組みの気持ちよさ”を丁寧に伝える。断定を避け、条件付きの表現（〜ことがある・理想的には）を守る。比喩はシンプルに、専門用語は先に概念→後から用語の順で導入。セクション間は問いかけでつなぎ、好奇心を持続させる。

## Thumbnail Direction

案A：片手でミウラ折りを“ひと引き”して一斉展開する瞬間の真上ショット＋大きな見出し「紙が機械になる」。案B：縦に引いた切り紙が横にも広がる瞬間を停止画で強調＋吹き出し「なぜ？」。案C：柔らかいグリッパーがトマトをやさしく掴む接写＋見出し「やさしく掴むロボット」。背景は白、文字は高コントラスト、ブランド要素なし。

## Production Notes

- 台本にない新しい事実・数字は追加しない。固有名詞・製品名・ミッション名は出さない。
- 使用素材は自社撮影またはオリジナルCG／図解に限定。ブランドロゴ・他社UIは映さない。
- 安全配慮：カッター使用は手元のみ・カッターマット必須・手袋の検討。空気圧は低圧で、顔を近づけない。加熱・薬品の実演はしない。
- 色覚多様性に配慮し、山折り／谷折りは色＋線種（実線／破線）で区別。
- ミウラ折りの“一自由度”や負のポアソン比は条件依存である旨を字幕かフットノートで毎回明示。
- 医療・宇宙・防災のカットは概念CGまたは模型のみ。リアル手術映像・特定施設映像は使用しない。
- 映像テンポ：実験手元はスローモーションを適宜挿入し、視認性を優先。数値表示は避け、相対比較で示す。
- 効果音は紙・空気・軽いクリックなど。BGMはミニマルで希望的、音量はナレーションを妨げないレベルに。
- 字幕とテロップは画面下端に統一し、主要図解と干渉しない配置にする。
- 撮影時は均一で柔らかい照明（影を最小化）。白背景の色温度を一定に保つ。
